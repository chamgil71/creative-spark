첫 대화: 프롬프트 설계

AI에게 무엇을 어떻게 말하느냐가 결과의 80%를 결정합니다. 같은 작업이라도 프롬프트 품질에 따라 AI의 결과물 수준이 완전히 달라집니다.

나쁜 프롬프트 vs 좋은 프롬프트

❌ 나쁜 프롬프트

엑셀 자동화 코드 만들어줘.

✅ 좋은 프롬프트

Python으로 엑셀 자동화 코드를 작성해줘.

입력: data.xlsx (A열:날짜, B열:부서명, C열:금액)

출력: summary.xlsx (부서별 월간 합계, 시트 1개)

조건:

  • 날짜 형식은 YYYY-MM-DD

  • 금액은 천 단위 콤마 표시

  • 주석은 한국어로

  • pandas와 openpyxl 사용

좋은 프롬프트의 5가지 구성 요소

🎭

역할 (Role)

"너는 Python 전문가야"

📋

맥락 (Context)

프로젝트 배경, 파일 구조

🎯

작업 (Task)

구체적으로 무엇을 할지

⚙️

조건 (Constraints)

언어, 형식, 제약사항

📤

출력 형식 (Format)

코드만, JSON, 한국어 등

3단계 요구사항 명세 기법

1

\"PDF 파일에서 예산 표를 추출해서 JSON으로 저장하는 Python 스크립트를 만들어줘.\"

2

입력 파일 경로, 컬럼 구조, 특이사항 / 출력 파일 형식, JSON 구조 예시

3

사용할 라이브러리, 금지 패턴, 에러 처리 방식, 주석 언어

SKILL.md — 재사용 가능한 AI 지시서

는 프로젝트의 핵심 맥락·규칙·용어를 한 파일에 정리해두는 방식입니다. 매번 같은 배경 설명을 반복하는 대신, 이 파일 하나를 첨부하면 즉시 프로젝트 맥락을 공유할 수 있습니다.

1. 프로젝트 개요

  • 목적:

과기부 PDF 예산문서를 파싱하여 SQLite DB에 저장

  • 언어:

Python 3.11

  • 현재 단계:

병합 셀 처리 버그 수정 중 (v15)

  • 금액 필드:

항상 정수(int) 타입 (float 금지)

  • HTTP 요청:

httpx 사용 (requests 금지)

  • 로깅:

logging 모듈 사용 (print 금지)

3. 현재 이슈

  • 3단계 이상 중첩 병합 셀 처리

  • PDF 파싱 기본 구조 완성

에이전트에게 복잡한 작업 위임하기

AI에게 "해줘"가 아니라 "맡길게"라고 말할 수 있을 때, 진짜 자동화가 시작됩니다. 복잡한 작업을 쪼개고, 검수하고, 안전하게 완성하는 전략을 다룹니다.

작업 분해 — 3가지 기준

1️⃣
1️⃣입출력 경계 명확히 ❌ 나쁜 분해 1단계: PDF 처리2단계: 데이터 저장 ✅ 좋은 분해 1단계: PDF → raw.json2단계: raw.json → structured.json3단계: structured.json → DB

2️⃣
결과를 파일로 저장

3가지 위임 패턴

📋

패턴 1 — 순차 위임

한 단계 완료 후 사람이 검수, 그 다음 단계를 지시. 가장 안전하고 기본적인 방식.

📐

패턴 2 — 계획 먼저

"실행하지 말고 계획만 먼저 보여줘"

→ 계획 승인 후 실행. 복잡·중요한 작업에 적합.

⚡

패턴 3 — 병렬 위임

독립적인 작업을 서브에이전트가 동시에 처리. PDF 10개를 5+5로 분할 병렬 처리.

중간 검수 포인트 — 언제 확인하나

검수 요청 프롬프트 패턴
1단계가 완료됐어. 진행 전에 다음을 확인해줘:
1. 추출된 총 레코드 수
2. 금액 필드의 최솟값·최댓값·null 개수
3. 예상과 다른 데이터 패턴이 있으면 알려줘
# 확인 후 내가 "계속"이라고 하면 2단계 진행

📐

계획 수립 후

방향이 맞는지 확인

1️⃣

1단계 완료 후

입력 데이터 검증

🔄

핵심 변환 후

데이터 구조 일치 확인

✅

전체 완료 후

요구사항 충족 확인

🚀

배포 전

운영 환경 동작 확인

복잡한 작업은

중간 결과를 파일로 보존

하는 세 원칙을 지키면 AI 위임의 실패율을 크게 낮출 수 있습니다.
