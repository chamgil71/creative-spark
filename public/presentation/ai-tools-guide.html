1. Claude 플랫폼 전체 구조

Claude는 Anthropic이 만든 AI 어시스턴트로, 접근 방식에 따라 세 가지 형태로 나뉜다. 목적에 맞는 진입 채널을 선택하는 것이 첫 번째 단계다.

3종 접근 채널 비교

웹 Claude (claude.ai)

브라우저에서 즉시 사용. 가장 쉬운 진입 방법으로 별도 설치 없이 로그인만 하면 된다. 아티팩트·프로젝트·커넥터 등 주요 기능을 모두 UI로 조작한다.

  • •설치 불필요
  • •아티팩트 미리보기
  • •프로젝트·커넥터 지원
  • •파일 첨부·음성 입력
Claude Desktop

Mac / Windows 전용 앱. 웹과 동일한 계정으로 사용하지만 MCP 서버 연결이 핵심 차이점이다. 로컬 파일 시스템, Notion, GitHub 등 외부 서비스를 직접 제어한다.

  • •MCP 서버 연결
  • •로컬 파일 접근
  • •파일 드래그앤드롭
  • •Alt+Space 글로벌 호출
Claude Code (CLI)

터미널에서 실행하는 에이전틱 코딩 도구. 파일을 읽고 수정하고, 테스트를 실행하며, Git을 조작하는 자율적 방식으로 동작한다. 개발자 전용 환경.

  • •npm install -g
  • •파일 자동 수정
  • •터미널 명령 실행
  • •CLAUDE.md 프로젝트 지침

아티팩트 (Artifacts) 활용법

대화 중 Claude가 생성한 독립 실행 가능한 결과물. 채팅창 오른쪽 별도 패널에 표시된다.

1
생성 요청

\"HTML로 할일 관리 앱 만들어줘\" 또는 \"SVG로 로고 그려줘\" 같이 구체적으로 요청한다. 아티팩트가 생성되면 자동으로 우측 패널이 열린다.

2
Preview / Code 전환

패널 상단 탭에서 Preview(실행 화면)와 Code(소스 코드)를 자유롭게 전환한다. 미리보기 상태에서 직접 클릭·입력하며 동작을 확인한다.

3
후속 수정 요청

\"버튼 색을 파란색으로 바꿔줘\", \"모바일에서도 잘 보이게 해줘\" 식으로 채팅에 계속 입력하면 아티팩트가 실시간 업데이트된다.

4
저장 및 공유

우측 상단 Copy 버튼으로 코드 복사, Download 버튼으로 파일 저장. Publish 버튼 클릭 시 공개 URL 생성 → 타인과 링크 공유 가능.

아티팩트 활용 예시

간단한 포트폴리오 페이지 뼈대 · 엑셀 데이터 붙여넣기 → 차트 즉시 생성 · 발표용 인터랙티브 슬라이드 · 계산기·게임 등 미니 웹앱

3. Claude Desktop

Desktop vs 웹 차이점

Claude Desktop만 가능한 것

MCP 서버 연결로 로컬 파일·외부 서비스를 직접 제어한다. Documents 폴더 읽기, Notion 페이지 쓰기, GitHub 저장소 조작 등 API 수준의 작업이 대화 한 줄로 가능해진다.

  • •로컬 파일 시스템 접근
  • •Notion · GitHub · Slack 제어
  • •SQLite DB 조회
  • •브라우저 자동화(Puppeteer)
웹 Claude만 가능한 것

별도 설치 없이 즉시 사용. Google 커넥터(Drive · Gmail · Calendar)는 웹 전용 OAuth 방식이라 Desktop에서는 MCP로 대체한다.

  • •설치 없이 즉시 사용
  • •Google 공식 커넥터
  • •Publish로 공개 URL 생성
  • •모바일 브라우저 접근

MCP 서버 등록 방법

상단 메뉴 → Claude → Settings (Mac: Cmd+,) → Developer 탭 → Edit Config 클릭 → claude_desktop_config.json 파일 편집 → 저장 후 Desktop 재시작.

4. Claude Code (CLI)

Claude Code 설치 및 실행 bash 
# 전역 설치 (Node.js 18 이상 필요)
npm install -g @anthropic-ai/claude-code
# 프로젝트 디렉토리로 이동 후 실행
cd my-project
claude
# 첫 실행 시 브라우저 인증창 자동 오픈 → Claude 계정으로 로그인

주요 슬래시 명령어

대화 관리

세션 컨텍스트와 토큰을 제어한다.

  • •/help : 명령어 전체 목록
  • •/clear : 컨텍스트 초기화
  • •/compact : 대화 요약 압축 (토큰 절약)
  • •/exit : 세션 종료
작업 실행

코드·앱을 직접 실행하고 검증한다.

  • •/run : 앱 실행 및 동작 확인
  • •/review : 현재 브랜치 PR 리뷰
  • •/code-review : 변경 코드 리뷰
  • •/verify : 변경사항 실제 동작 검증
설정 · 자동화

모델·권한·반복 작업을 관리한다.

  • •/config : 모델·테마 설정 변경
  • •/memory : 기억 파일 편집
  • •/schedule : 반복 작업 예약
  • •/fast : Opus 고속 모드 전환

CLAUDE.md (프로젝트 지침 파일)

# 프로젝트 지침
## 기술 스택
- Python 3.11, FastAPI, PostgreSQL
- 테스트: pytest / 린트: ruff, mypy
## 코딩 규칙
- 모든 함수에 타입 힌트 필수
- 커밋 메시지: feat: / fix: / docs: 접두어 사용
- 영어 변수명, 주석은 한국어
## 절대 금지
- print() 대신 logging 사용
- 하드코딩된 API 키 금지 (.env 파일 활용)
- 테스트 없는 기능 추가 금지

Claude Code 에이전틱 동작 흐름

1
파일 탐색 및 구조 파악

프로젝트 디렉토리를 자동 스캔하여 package.json, tsconfig.json 등을 읽고 기술 스택을 파악한 뒤 수정 대상 파일을 열람한다.

2
정밀 부분 패치 (Diff Patch)

전체 파일을 재작성하지 않고, 수정이 필요한 라인만 정확히 포인팅하여 오리지널-업데이트 블록 구성으로 코드 일부만 안전하게 교체한다.

3
빌드 및 테스트 자동 실행

수정된 소스가 컴파일에 실패하지 않는지 npm run build, pytest 등을 CLI에서 직접 실행하여 통과 여부를 검증한다.

4
자율 디버깅 루프

에러 로그가 터미널에 수신되면 원인을 스스로 추적하여 재패치 후 재검증하는 피드백 루프를 완수한다. 사람이 개입하지 않아도 된다.

5. Claude 활용 전략

지침 작성의 4요소

효과적인 지침은 역할 + 맥락 + 형식 + 제약 네 가지를 모두 포함한다.

누구로 행동할지
👤역할 (Persona)

\"너는 10년 경력의 UX 라이터야. 기술 문서를 일반인도 쉽게 읽도록 바꿔줘.\"

배경 정보
🗺️맥락 (Context)

\"우리 서비스는 40~60대 직장인 대상 건강관리 앱이야. 사용자들은 IT에 익숙하지 않아.\"

출력 형태
📄형식 (Format)

\"결과는 마크다운 표로 정리하고, 각 항목에는 예시를 하나씩 포함해줘.\"

하지 말아야 할 것
🚫제약 (Constraint)

\"전문 의학 용어는 쓰지 마. 영어는 괄호 안에만 병기해.\"

지침 만드는 실전 흐름

1
먼저 자연어로 질문해보기

원하는 작업을 평소 말하듯 자유롭게 입력한다. 결과가 마음에 안 들면 \"더 짧게\", \"다른 형식으로\" 식으로 방향을 조정한다.

2
만족스러운 결과 확보

원하는 수준의 결과가 나오면, \"방금 내가 준 요청을 재사용 가능한 지침으로 정리해줘\"라고 요청한다. Claude가 구조화된 지침 텍스트를 생성해준다.

3
지침 등록

정리된 지침을 프로젝트 지침(웹 Claude) 또는 CLAUDE.md(Claude Code)에 붙여넣는다. 이후 모든 대화에 자동 적용된다.

Claude 루틴 (정기 자동화)

Claude Code의 /schedule 기능으로 반복 작업을 클라우드 에이전트로 등록. PC가 꺼져 있어도 자동 실행된다.

1
/schedule 입력

Claude Code 터미널에서 /schedule 입력하면 루틴 등록 대화가 시작된다.

2
작업·주기 지정

작업 내용(자연어로 설명)과 실행 시각(cron 표현식 또는 \"매일 오전 8시\"), 반복 주기를 지정한다.

3
클라우드 등록 완료

/schedule list로 등록된 루틴 목록 확인. /schedule delete로 삭제. 등록 후에는 로컬 PC 없이 클라우드에서 자동 실행.

루틴 활용 예시

매일 오전 8시: 뉴스 헤드라인 수집 → 주요 일정 정리 → 이메일 발송 자동화

매주 월요일 주간 보고 초안 생성매일 밤 GitHub 이슈 요약정기 데이터 백업

11. AI 협업 레슨런

실전 AI 협업에서 시행착오를 거쳐 정리한 14가지 핵심 원칙. 설계·역할 분리부터 프롬프트 기술까지.

레슨런 — 설계·구조 원칙

레슨런 #1
🎯역할 분리

AI에게 설계를 위임하지 않는다. 설계는 사람이, 구현은 AI가.

레슨런 #3
🔌파이프라인 설계

각 단계의 입출력 형식을 계약처럼 명시. 인터페이스가 바뀌어도 전체가 무너지지 않는다.

레슨런 #4
⚙️설정·코드 분리

변하는 것(색상·문구·API)은 설정 파일로, 변하지 않는 것(로직·구조)은 코드로.

레슨런 — 기록·자산 원칙

레슨런 #2
📋실패 기록

오류 메시지와 실패한 시도를 반드시 기록한다. AI는 이전 대화를 기억하지 못한다. 오류 로그가 다음 세션의 출발점이 된다.

레슨런 #5
💾원본 보존

복사 → 검증 → 삭제 순서를 지킨다. AI가 파일을 수정하기 전 항상 원본을 복사한다.

레슨런 #7
📦자산 축적

대화는 흘러가지만 파일은 남는다. 성공한 프롬프트와 패턴을 SKILL.md에 저장해 다음 프로젝트에서 재사용한다.

레슨런 #10
🗂️미완성 기록

완성보다 기록이 먼저다. 막힌 지점·다음 할 일을 남겨두면 재개가 쉬워진다.

레슨런 — 실행 원칙

레슨런 #8
😤고통 우선

자동화는 실제로 고통스러운 부분부터 시작한다. 고통이 명확할수록 자동화 범위도 명확해진다.

레슨런 #9
🌱최소 인프라

가장 작은 v1으로 시작해 실제로 동작하는 것을 확인한 뒤 점진적으로 확장한다.

레슨런 #13
⚡즉시 시작

완벽한 설계를 기다리는 것이 가장 비싼 실수. 불완전한 v1이라도 실행해야 실제 문제를 알 수 있다.

레슨런 — 소통·프롬프트 원칙

12. 설계와 구조화

단편적 질답에서 구조화된 재사용성으로 — 지침이 생산성을 결정한다.

단편적 질답 vs 구조화된 접근

단편적 질답 방식

매번 맥락을 처음부터 설명한다. 대화가 끝나면 패턴이 사라진다. AI에게 설명하는 시간이 전체 작업의 절반을 차지한다.

  • •매 세션 맥락 재설명 필요
  • •결과 품질이 일관되지 않음
  • •다른 팀원이 재현 불가
  • •패턴이 축적되지 않음
구조화된 재사용 방식

SKILL.md와 시스템 프롬프트로 맥락을 파일에 저장한다. 한 번 만든 지침이 모든 세션에 자동 적용된다.

  • •지침 파일 한 번 작성으로 영구 재사용
  • •팀원 모두가 동일 품질 재현
  • •프로젝트 간 이식 가능
  • •점진적으로 지침 품질 향상

SKILL.md — 재사용 가능한 지침 구조

SKILL.md 템플릿 (markdown)
12345678910 
[스킬 이름] 주간 업무 보고서 생성
[언제 사용] 매주 금요일 업무 정리 시
[입력] 이번 주 완료한 업무 목록 (불릿 형식)
[출력 형식]
- 제목: "YYYY-WW 주간 업무 보고"
- 섹션: 완료 / 진행중 / 계획
- 길이: A4 1장 이내
- 어조: 공식 보고서 (존댓말)
[예시 입력/출력]
... (실제 예시 포함)

설계(지침)의 힘 — 4가지 효과

⏱️맥락 파악 시간 단축

AI가 프로젝트 배경을 묻지 않아도 된다. 지침 파일이 맥락을 대신 전달한다.

🔁반복 수정 횟수 감소

출력 형식과 제약 조건이 미리 정의되어 있어 첫 결과물의 품질이 높아진다.

👥팀 전체가 동일 품질

지침 파일을 공유하면 누가 써도 같은 수준의 결과를 얻는다. 개인 역량에 의존하지 않는다.

🔄프로젝트 간 이식

잘 만든 SKILL.md는 다른 프로젝트에 그대로 가져다 쓸 수 있다. 지침이 곧 자산이다.

13. 서비스 연결과 조립

도구는 연결될수록 강해진다. 각 도구가 가장 잘 하는 한 가지만 맡기고 결과를 이어 붙인다.

연결 방식 선택 가이드

API · MCP (개발자 방식)

Claude와 외부 서비스를 코드나 MCP 서버로 직접 연결. 완전한 제어가 가능하고 복잡한 로직을 구현할 수 있다.

  • •API 직접 호출: 완전한 제어, 코드 필요
  • •MCP 서버: Claude와 서비스 직접 연동
  • •Claude Code 환경에서 즉시 활용 가능
웹훅 · 노코드 (빠른 방식)

n8n, Zapier, Make 등 노코드 플랫폼으로 이벤트 기반 자동화 구성. 코드 없이 시각적으로 파이프라인을 연결한다.

  • •웹훅: 이벤트 기반 자동 트리거
  • •n8n: 자기 호스팅 노코드 자동화
  • •비개발자도 빠른 프로토타이핑 가능

조립의 핵심 원칙

1
고통스러운 지점부터

막연한 자동화가 아니라, 실제로 반복되고 고통스러운 부분을 먼저 자동화한다.

2
인터페이스 계약 먼저

각 단계의 입출력 형식을 미리 정의한다. 중간 단계가 바뀌어도 전체가 무너지지 않는다.

3
단계별 검증

전체를 한꺼번에 연결하지 않는다. 한 단계씩 동작을 확인하며 이어 붙인다.

4
실패 알림 + 로그

자동화가 조용히 실패하는 것이 최악이다. 어느 단계에서 문제가 생겼는지 추적 가능하게 로그를 남긴다.

백문이 불여일견, 백견이 불여일타, 백타가 불여일문 !!

AI 도구에 대한 설명은 읽는 것만으로 감각이 생기지 않는다. 내 업무에서 가장 고통스러운 부분이 무엇인지 묻는 것에서 시작하라. 그 답이 나왔을 때, 이 모든 도구와 원칙이 비로소 의미를 가진다.

설계는 사람이, 구현은 AI가지침이 정밀할수록 결과가 달라진다도구는 연결될수록 강해진다
