Playwright는 어떤 도구인가

Playwright는 코드를 통해 크롬, 엣지, 파이어폭스 등 웹 브라우저를 직접 조종하는 라이브러리입니다. 프론트엔드 개발자들은 자신이 만든 화면이 잘 작동하는지 테스트(E2E)하는 용도로 쓰며, 자동화 개발자들은 화면에 동적으로 그려지는 뉴스, 부동산, 주식 데이터를 막힘없이 긁어오는(Crawling) 최강의 도구로 사용합니다.

한눈에 보는 핵심 기능

⏳
자동 대기 (Auto-wait)

요소가 화면에 나타날 때까지 알아서 기다려주어, `sleep(5)` 같은 불안정한 지연 코드를 쓸 필요가 없습니다.

🕵️‍♂️
동적 렌더링 완벽 처리

React나 Vue로 만들어져 클릭하거나 스크롤해야만 데이터가 나오는 최신 웹사이트의 데이터를 쉽게 추출합니다.

🎥
트레이스 및 화면 녹화

크롤링 중 에러가 나면, 스크립트가 어느 시점에서 멈췄는지 스크린샷과 비디오로 녹화해 줍니다.

🥷
브라우저 컨텍스트

브라우저를 아예 새로 띄우지 않고 가벼운 '시크릿 탭' 형태로 여러 개를 동시에 열어 수집 속도를 극대화합니다.

이런 상황에서 압도적인 위력을 냅니다

수집
📊 부동산/경매 데이터 수집가

지도를 이리저리 이동하고 팝업을 클릭해야만 볼 수 있는 복잡한 경매 사이트의 데이터를 매일 추출해야 할 때.

자동화
📰 뉴스 애그리게이터 제작

매일 아침 여러 언론사를 돌며 특정 키워드의 기사 제목과 본문을 긁어내 리포트로 만들 때.

품질
🧪 프론트엔드 QA

내가 만든 Vercel 배포 사이트에 "로그인 후 장바구니에 담기" 로직이 잘 도는지 매번 확인하기 귀찮을 때.

시작하기: 파이썬 기반 데이터 추출

BeautifulSoup이나 Requests로는 한계가 있는 동적 웹페이지를 공략하는 방법입니다.

기본 크롤링 흐름 (Python 기준)

1
패키지 및 브라우저 설치

터미널에서 `pip install playwright` 후, `playwright install`을 쳐서 구동용 브라우저를 다운로드합니다.

2
브라우저 실행 설정

백그라운드에서 조용히 실행할지(Headless), 눈으로 보면서 디버깅할지(Headed) 설정합니다.

3
페이지 이동 및 조작

`page.goto("URL")`로 이동 후, `page.locator(".btn-search").click()` 처럼 요소를 특정해 액션을 취합니다.

4
데이터 추출 (평가)

`page.eval_on_selector_all()` 등을 사용하여 원하는 태그 내부의 텍스트(텍스트 콘텐츠)나 속성 값을 가져옵니다.

5
파일 저장 및 종료

추출한 배열을 JSON이나 CSV로 가공하여 저장하고, 열어둔 브라우저 컨텍스트를 깔끔하게 종료합니다.

3

크롤링 차단(봇 탐지) 우회 팁

User-Agent 커스텀

스크립트 실행 시 실제 모바일 기기나 특정 데스크톱 브라우저인 것처럼 User-Agent 값을 속여 접속합니다.

차단 방지의 기본
네트워크 가로채기

굳이 HTML을 긁지 않고, 브라우저의 네트워크 탭으로 오가는 XHR/Fetch API 응답(JSON)만 깔끔하게 빼냅니다.

속도/정확도 최고 수준
Stealth 플러그인

매우 강력한 봇 방어(Cloudflare 등)가 걸려있다면, Playwright-stealth 모듈을 엮어 봇이 아닌 척 우회합니다.

고급 자동화 기술
4

마무리 체크리스트

스크립트 배포 전 최적화

  • 불필요한 이미지나 폰트, CSS 로딩을 차단(Abort)하여 스크립트 실행 속도를 극대화했는가?
  • 선택자(Selector)가 너무 길고 복잡해 사이트 디자인이 살짝만 바뀌어도 깨질 위험이 없는가? (Id나 Data 속성 권장)
  • 크롤링 중 예기치 못한 팝업(광고, 공지사항)이 떴을 때 이를 무시하거나 닫아주는 예외 처리를 했는가?

💡 TIP: 선택자(Selector)를 찾는 데 시간을 낭비하지 마세요. 터미널에 playwright codegen <사이트주소>를 치면, 사용자가 클릭하고 입력하는 모든 행동을 즉석에서 파이썬/JS 코드로 자동 기록해 줍니다.
