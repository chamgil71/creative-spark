# 서론: AI가 스스로를 만든다는 것

Anthropic은 현재 AI 개발의 점점 더 많은 부분을 AI 시스템 자체에 위임하고 있으며, 이것이 작업 속도를 높이고 있습니다. 이 흐름이 충분히 진행되고 충분한 컴퓨팅 파워가 주어진다면, 결국 **완전히 자율적으로 자신의 후속 버전을 설계·개발하는 AI**, 즉 **재귀적 자기개선(Recursive Self-Improvement; RSI)** 이 가능해진다는 것이 이 글의 핵심 전제입니다.

아직 그 단계에 도달하지 않았고, RSI가 불가피한 것도 아니지만, 대부분의 기관이 준비하는 것보다 훨씬 일찍 올 수 있습니다.

::: takeaway-banner
- icon: "🔄"
  title: "Recursive Self-Improvement"
  desc: "스스로 자신의 성능을 올리고 코딩하는 AI의 탄생은, 단순히 생산성이 늘어나는 것 이상의 복리적 기술 폭발로 이어질 수 있습니다."
:::

# 2장: Anthropic 내부 증거 (Internal Evidence)

이 챕터가 이 글의 핵심입니다. 공개 벤치마크만으로는 AI가 **AI 개발 자체를 얼마나 가속하고 있는지** 알 수 없기 때문에, Anthropic 연구소의 내부 데이터를 공개했습니다.

::: summary-box
- title: "핵심 요약"
  desc: "코드 작성부터 품질 검토, AI 모델 최적화 실험 및 제안, 장기적인 연구 방향 결정에 이르기까지 AI 연구개발 전 영역에 걸쳐 AI가 직접 개입하고 있습니다."
  meta: "코드 작성 80% 돌파 | 생산성 8배 향상 | 초인적 최적화 속도"
:::

## 2-1. 코드 작성 (Code Authoring)

2026년 5월 기준, Anthropic 코드베이스에 병합되는 코드의 **80% 이상이 Claude가 작성**한 것입니다. Claude Code가 출시되기 전(2025년 2월 이전)에는 이 수치가 한 자릿수였습니다.

::: stat-grid cols=2
- icon: "80%+"
  title: "Claude 작성 코드 비중"
  desc: "현재 Anthropic의 전체 서비스 및 모델 리서치 코딩 코드 중 8할 이상이 AI가 자율 합성함"
- icon: "8배"
  title: "엔지니어 생산량 증가"
  desc: "자율 실행 및 장시간 작업 에이전트 도입에 힘입어 엔지니어 1인당 코드 병합량이 2024년 대비 8배 상승"
:::

*실제 사례*: 2026년 4월, Claude는 800개 이상의 수정을 완료해 특정 유형의 API 오류를 1,000분의 1로 줄였습니다. 이 작업을 사람이 했다면 4년이 걸렸을 것으로 담당 엔지니어가 추산했습니다.

---

## 2-2. 코드 품질 (Code Quality)

Claude가 코드 검토자 역할도 맡게 되었으며, 코드베이스의 모든 변경 사항을 사전 검토하는 자동 리뷰 시스템을 도입했습니다.

::: alert-box tip
- title: "자동 리뷰 시스템의 효과"
  desc: "과거 claude.ai 장애(인시던트)의 약 3분의 1이 이 AI 사전 리뷰 시스템이 있었다면 프로덕션 반영 전에 미리 잡혔을 것으로 조사되었습니다."
:::

*인간과의 품질 비교*: Claude가 작성한 코드의 품질은 2025년 말 인간 작성 코드보다 다소 낮았으나, 현재는 대략 동등한 수준이며, 1년 안에 인간보다 우수해질 것으로 예상됩니다.

# 3장: Anthropic의 미래 업무 형태

인간과 AI가 작성한 코드의 품질이 완전히 동등해지면, 인간은 직접 코드 작성을 멈추고 검토만 하게 될 것입니다. 하지만 AI의 생성 속도가 압도적일 경우 인간의 검토 자체가 병목이 됩니다.

::: takeaway-banner
- icon: "🎭"
  title: "연구 취향 (Research Taste)"
  desc: "어떤 문제가 진정으로 중요한지, 어떤 결과를 신뢰할지, 언제 도전을 멈출지 결정하는 인간 고유의 영역이 최후의 보루가 될 것입니다."
:::

::: columns-grid cols=2
- icon: "🔄"
  title: "점진적 개선의 힘"
  desc: "AI의 발전은 '유레카' 순간보다는 실험 확장, 원인 파악, 디버깅, 재수행의 반복이었으며, 이는 AI 에이전트가 가장 뛰어난 능력을 발휘하는 도메인입니다."
- icon: "📈"
  title: "연구 취향의 자동화"
