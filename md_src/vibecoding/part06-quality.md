# 오류 처리와 디버깅 협업

AI가 틀렸을 때가 진짜 실력이 드러나는 순간입니다. 오류를 빠르게 해결하는 것은 기술이 아니라

피드백 루프를 잘 설계하는 것

입니다.

### 오류 4가지 유형

해결 쉬움

유형 1 — 문법·실행 오류

코드가 즉시 멈추고 오류 메시지 출력. 오류 메시지 전체를 AI에게 붙여넣으면 대부분 해결.

해결 어려움

유형 2 — 논리 오류

코드는 실행되지만 결과가 잘못됨. 예상값과 실제값을 함께 제시해야 AI가 원인을 파악.

예방 가능

유형 3 — AI 환각 오류

존재하지 않는 함수·라이브러리 사용. 실행 전 "이 메서드가 실제로 있는지 확인해줘"로 예방.

SKILL.md 해결

유형 4 — 컨텍스트 소실

AI가 이전 결정을 무시하고 다른 방식 사용. SKILL.md 재첨부 + 구체적 위반 항목 명시로 해결.

❌ 잘못된 오류 보고:

"실행이 안 돼요. 오류가 났어요."

✅ 올바른 오류 보고:

"다음 오류가 발생했어. 수정해줘:

Traceback (most recent call last):

File "json_structurer.py", line 847, in fill_merged_cells

prev_value = prev_row[col_idx]

IndexError: list index out of range

실행 환경: Python 3.11

입력: /output/raw/budget_2026_p042.json"

### 반복 실패 루프 탈출 — 3단계

1

현재 상태 요약 요청

"지금까지 시도한 방법과 결과를 요약해줘. 아직 시도하지 않은 접근법 3가지를 제안해줘."

2

문제 재정의

"처음부터 다시 생각해보자. 현재 방식 말고 완전히 다른 방법으로 접근한다면?"

3

단순화 (최소 동작 버전)

"현재 코드를 버리고, 병합 셀 처리 없이 단순 표만 처리하는 최소 버전부터 만들어줘."

### 예방적 디버깅 — 방어 코드 요청 패턴

::: icon-grid cols=2
- icon: "🛡️"
  title: |
    🛡️방어 코드 요청 프롬프트
     
     "parse_table() 함수 작성 시 포함할 것:
     ① 입력이 None이거나 빈 리스트면 빈 리스트 반환
     ② 행의 열 수가 다를 수 있음을 고려
     ③ 금액 파싱 실패 시 None 저장 (예외 전파 금지)
     ④ 오류 행은 로그 기록 후 건너뜀 (전체 실패 금지)"
- icon: "🧪"
  title: |
    🧪TDD — 테스트 먼저 작성
     
     "parse_amount() 테스트 케이스 먼저:
     '1,234,567' → 1234567
     '(△123,456)' → -123456
     '-' → None
     '' → None
     pytest 형식으로 작성 후 통과하는 함수 구현해줘."
:::

::: alert-box warn
- title: "⚠️ 3중 안전망:"
  desc: |
    Git 커밋 (AI 작업 전 항상) → 원본 파일 불변 (chmod 444) → DB 스냅샷 (cp budget.db budget.db.backup.YYYYMMDD). 이 세 가지만 지켜도 모든 상황에서 복구 가능합니다.
:::

💡

Key Takeaway

AI 디버깅의 핵심은

오류 메시지 전체 붙여넣기, 예상값과 실제값 동시 제시, 3회 반복 실패 시 루프 탈출 질문

이며, Git 커밋과 원본 파일 불변 원칙이 모든 상황의 안전망입니다.

# 재사용 가능한 자산 만들기

좋은 자산은 한 번 만들고 열 번 씁니다. AI 협업에서 쌓인 경험을

다음 프로젝트의 시작점을 높이는 영구 자산

으로 만드는 방법입니다.

### 5가지 자산 유형

📝

SKILL.md

AI 지시서

프로젝트별 규칙

💬

프롬프트 템플릿

검증된 프롬프트

Obsidian 아카이브

🧩

코드 스니펫

재사용 함수

GitHub snippets/

🔄

n8n 워크플로우

템플릿 JSON

내보내기/가져오기

🚀

프로젝트 스타터

GitHub Template

즉시 시작 가능

### 자산 업데이트 트리거 — 언제 업데이트하나

| 신호 | 행동 |
| --- | --- |
| 같은 버그를 두 번째 만났을 때 | SKILL.md "금지 사항"에 추가 |
| "이 코드 어디서 보지 않았나?" | snippets/에 저장 |
| 새 팀원이 같은 질문을 할 때 | Notion 팀 위키에 답변 추가 |
| AI가 같은 잘못된 패턴을 반복 | SKILL.md 금지 사항에 추가 |
| 프로젝트 완료 | 회고 문서 + 핵심 자산 아카이빙 |
