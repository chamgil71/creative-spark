# 1. 도입 (Introduction)

## 터미널 기반 AI 에이전트의 등장 배경
기존의 GUI 기반 웹 채팅(Chat)이나 단순 인라인 자동완성(Autocomplete)의 한계를 극복하기 위해 터미널 에이전트가 등장했습니다. 개발자가 마우스를 손에 쥐지 않고, 터미널 환경을 떠나지 않은 채 자연어로 개발 프로세스 전체를 제어하는 워크플로우가 대세로 자리 잡았습니다. 단순 코드 추천을 넘어 파일 읽기/쓰기, 셸 명령어 실행, 테스트 런, Git 커밋 및 PR 생성까지 스스로 수행하는 에이전틱(Agentic) 환경을 제공합니다.

  color: "#10A37F"
:::

# 2. Claude Code: CLI의 정석과 정밀 제어

## 방식 개요 및 실행 예시
Anthropic이 CLI 환경에 최적화하여 내놓은 도구로, 개발자의 의도를 가장 정교하게 파악하고 로컬 파일 시스템을 안전하게 제어합니다. 의도 분석 후 관련 파일을 탐색 및 읽고, 리팩토링 계획 수립, 파일 일괄 수정, 터미널 테스트 실행까지 유기적으로 이어집니다.

::: cmd-box
title: Claude Code 실행 인터페이스
meta: bash
desc: |
  $ claude
  # Claude Code 세션 진입 후 자연어로 명령
  > "이 프로젝트의 기존 Express 라우터들을 전부 아키텍처 패턴에 맞춰 Controller/Service 레이어로 분리하고, 깨지는 테스트가 없는지 npm test로 검증해줘."
:::

# 3. Antigravity 2.0: 대규모 컨텍스트와 멀티 에이전트

## 방식 개요 및 실행 예시
Google Gemini 계열의 초거대 컨텍스트 윈도우를 무기로 삼아, 프로젝트 전체 구조를 통째로 컨텍스트에 올린 채 작업하는 멀티 에이전트 오케스트레이터입니다. 대형 목표(Goal)를 접수하면 백엔드 에이전트와 프론트엔드 에이전트로 태스크를 분할하여 병렬 처리합니다.

::: cmd-box
title: Antigravity 2.0 자율 목표 설정
meta: bash
desc: |
  $ antigravity --goal "현재 프론트엔드 API 호출부를 최신 백엔드 스펙에 맞게 전면 수정하고, 백엔드 컨테이너와 연동하여 E2E 테스트까지 완료해"
:::

# 4. Codex: 실행 효율과 엔지니어링 자동화

## 방식 개요 및 실행 예시
OpenAI의 진화된 엔지니어링 에이전트로, 철저하게 '목표 달성'과 '비용/토큰 효율성'에 초점이 맞춰져 있습니다. 스스로 가상 샌드박스를 구축하거나 CI/CD 파이프라인과 완벽하게 맞물려 독립된 이슈를 해결하는 데 특화되어 있습니다.

::: cmd-box
title: Codex 이슈 해결 및 PR 자동화
meta: bash
desc: |
  $ codex task "Issue #142번 버그 해결하고, 관련 유닛 테스트 작성한 뒤 origin/feature/142 브랜치로 PR 올려줘"
:::

## 나의 실무 활용 경험 공유
:::feature-grid
- icon: 📦
  title: 완성형 PR 생성기
  tag: 자동화 위임
  desc: 단순히 코드만 고치는 게 아니라 변경 로그, 영향도 평가, 테스트 결과 텍스트까지 포함된 고품질의 Pull Request를 알아서 생성.

:::

::: compare-split
- title: 🟥 Claude Code (Anthropic)

  note: 대형 아키텍처 및 대형 기능 추가 추천
:::
