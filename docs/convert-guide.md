# 📖 가이드 변환기 & 에디터 (Converter) 사용설명서

Creative Spark 프로젝트에 새롭게 추가된 **가이드 변환기 및 에디터(Converter)**는 개발자와 가이드 작성자가 마크다운을 시각적으로 작성하고, 실시간 미리보기를 통해 검증하며, 원하는 배포용 파일 형식(HTML, 횡형 슬라이드 HTML, PPTX 파워포인트)으로 로컬에 일제히 저장할 수 있는 통합 GUI 도구입니다.

---

## 🛠️ 주요 기능 및 활용법

### 1. 실시간 Split-Pane(가변폭 분할 화면) 편집 및 미리보기
* **IDE 스타일 편집 환경**: 좌측의 마크다운 소스 편집기에서 글을 쓰거나 숏코드를 편집하고 `[미리보기 반영 (확인)]` 버튼을 클릭하면, 우측 화면에 실시간으로 빌드 결과가 렌더링됩니다.
* **미리보기 모드 전환**:
  * **가이드 (A4)**: 세로형 반응형 웹북 문서 레이아웃으로 미리보기를 띄웁니다.
  * **슬라이드 (16:9)**: Marp 호환성 및 PPT 슬라이드 규격(16:9)에 맞추어 횡으로 한 장씩 넘겨 보는 프레젠테이션 화면으로 띄웁니다.
* **디바이스 뷰포트 시뮬레이션**: 우측 툴바에서 **데스크탑, 태블릿, 모바일** 버튼을 클릭하면 해당 규격에 맞게 뷰포트 너비가 조절되어 반응형 레이아웃을 실시간 모니터링할 수 있습니다.

### 2. 실시간 디자인 테마 관리자 (Theme Manager)
상단 헤더의 `🎨 테마 설정` 버튼을 클릭하면 화면 오른쪽에서 디자인 테마 매니저 드로어가 나타납니다.
* **테마 종합 조회**: 현재 시스템에 등록된 모든 테마의 브랜드 HSL 색상(메인, 어두운바, Dark, Deep, Light, Mid, 배경, 테두리) 및 Web/PPTX 폰트 설정을 표 형태로 한눈에 볼 수 있습니다.
* **테마 생성 / 수정 / 삭제**:
  * 컬러피커를 통해 테마의 색상을 시각적으로 변경하거나 폰트 설정을 바꿀 수 있습니다.
  * 새로운 테마 ID를 입력해 디자인 프리셋을 무제한으로 추가할 수 있습니다.
* **서버 저장**: 수정한 테마들은 하단의 `서버에 최종 저장하기`를 누르면 서버의 `config/styles.json` 파일에 즉시 쓰여 반영되므로, 영구적인 보관 및 빌드 연동이 가능합니다.

### 3. 유연한 로컬 저장 및 빌드 파이프라인
에디터 좌측 하단의 **확정 생성 및 로컬 저장 옵션**을 통해 파일 저장 및 자동 컴파일 파이프라인을 제어합니다.
* **저장 위치 설정**:
  * **단독 파일 생성**: 가이드 허브 목록에 문서를 엮지 않고, 사용자가 지정한 로컬 출력 폴더(예: `dist-pptx`)에 변환된 결과물들만 단독으로 추출해 냅니다.
  * **가이드 허브 추가**: 원천 마크다운 문서를 `md_src/guides/` 폴더에 자동으로 파일로 기록하고 가이드 목록 인덱스(`guides.json`)를 동적으로 갱신하여, 가이드 허브 메인 페이지에 즉시 노출되도록 빌드합니다.
* **생성 파일 다중 선택**: 체크박스를 통해 **HTML(세로), 횡HTML(슬라이드), PPTX(파워포인트)** 형식을 원하는 대로 다중 선택하여 한 번의 클릭으로 동시 생성할 수 있습니다.

## 📽️ 마크다운 작성 꿀팁 (Advanced Guide Tips)

### 1. 표지 제목 여러 줄(두 줄 이상)로 작성하기
마크다운 최상단 Frontmatter의 `title`에 스칼라 지시자 `|`를 사용하면 표지 제목을 원하는 위치에서 여러 줄로 깔끔하게 줄바꿈할 수 있습니다. HTML 및 PPTX 변환 엔진이 텍스트 박스 높이와 아래 부제목의 Y좌표 오프셋을 자동 계산하므로 글자가 절대 겹치지 않습니다.

```yaml
---
title: |
  중력을 거슬러
  AI로 한계를 넘어서는 개발 생산성의 새로운 기준
subtitle: "Vite + PPTX 자동화 변환 가이드북"
style: "custom-blue"
---
```

### 2. 한 화면에 2단 워크플로우 배치 및 화살표 바운스
슬라이드(16:9) 한 장에 두 개 이상의 숏코드(`workflow-flow`)를 위아래로 나란히 놓고, 그 사이에 수직 화살표를 배치하고 싶다면 중간 구분선(`---`)을 넣지 않고 아래와 같이 줄바꿈 및 단독 화살표 기호(`↓`)를 한 줄로 적어주시면 됩니다.

* **동작 원리**: 변환 엔진이 단독 화살표 행을 감지하면 자동으로 **정중앙 정렬 및 통통 튀는 바운스 애니메이션이 가미된 화살표 세퍼레이터**로 치환 렌더링해 줍니다.
* 슬라이드 모드 전용 `.workflow-flow` 크기 슬림화가 적용되어 높이가 겹치거나 초과하지 않습니다.

```markdown
# 통합 AI 개발 워크플로우

::: workflow-flow
- title: "💡 아이디어 발생"
- title: "📓 Obsidian 기록"
:::

↓  <!-- 👈 단독 화살표 기호를 적으면 자동으로 중앙 정렬 및 바운스 세퍼레이터가 됩니다! -->

::: workflow-flow
- title: "💻 VS Code 개발"
- title: "💻 Claude Code 구현"
:::
```

### 3. 좌우로 시원한 표(Table) 활용
* **슬라이드 모드**: 표가 화면을 널찍하게 사용해 정보를 시원하게 노출하도록 가로 폭이 `max-width: 96%` 및 중앙 배치(`margin: 20px auto`)로 확장 튜닝되었습니다. 짝수 행 얼룩말 무늬와 호버 효과, 체크표시 자동 초록색 강조 효과가 탑재되어 있습니다.
* **세로형 문서 모드**: 가이드북 문서 본문 안에서는 투명 배경 대신 차분한 그레이 보더와 헤더 톤 디자인으로 구성된 세로 문서용 테이블 스타일이 본문 카드 폭에 딱 맞추어(100% 폭) 세련되게 적용됩니다.

## 🏃 실행 및 빌드 가이드

### 1. 가이드 변환기만 단독 실행하기 (가장 추천)
가이드 허브 메인 대시보드나 React 관련 의존성 없이, 오직 에디터 창만 띄우고 가볍게 문서를 편집/변환하고자 할 때 사용하는 모드입니다. 저장 위치 설정 등의 복잡한 UI가 가려지고 로컬 폴더 저장 기능만 심플하게 기동됩니다.

```bash
npm run dev:converter
```
* 위 명령을 실행하면 자동으로 브라우저에서 `http://localhost:8080/converter.html` 경로로 독립 에디터가 열립니다.

### 2. 가이드 허브와 연동하여 임베드 모드로 실행하기
작성한 문서를 전체 가이드 허브 목록에 포함하고, 통합 Standalone 패키징까지 일체 구동하고자 할 때 사용합니다.

```bash
# 로컬 개발 서버 기동 (메인 홈 및 에디터 동시 기동)
npm run dev
# → http://localhost:8080 에서 메인 대시보드 접속 후 상단의 '변환기 열기' 클릭
```

### 3. 독립형 에디터 빌드
독립형 가이드 변환기 HTML 페이지(`converter.html`)와 번들 에셋만 따로 정적으로 빌드하여 배포하고자 할 때 사용합니다.

```bash
npm run build:converter
# → dist/converter.html 및 관련 에셋 번들 생성
```
