## 1. 필수 Frontmatter 구성 지침

모든 생성 파일 최상단에는 아래 양식의 YAML frontmatter가 정확히 기재되어야 합니다. 누락되거나 틀려서는 안 됩니다.

```yaml
---
title: 도구 이름 (예: 클로드 코드)
subtitle: 도구의 용도를 명확히 알 수 있는 한 줄 요약 문구 (예: "터미널에서 AI와 짝 코딩을 시작해봐요!")
logo: 도구에 어울리는 대표 이모지 1개 (예: 💬)
badge: 칩에 표기할 강조 라벨 (예: 🎉 완전 초보자 맞춤 가이드)
style: [테마스타일키]       # 아래 10개 색상 프리셋 테마 중 가장 성격에 맞는 1개 지정
# [선택 사항] 문서 전역 디자인 오버라이드
fontFace: "Malgun Gothic"   # 개별 문서용 기본 폰트 오버라이드
titleSize: 22               # 개별 문서용 슬라이드 타이틀 크기 일괄 강제 설정 (기본 28pt)
bodySize: 12                # 개별 문서용 슬라이드 본문 크기 일괄 강제 설정 (기본 14pt)
heroCta:
  label: 시작하기           # 스크롤 이동 대상 버튼 라벨
  url: #what               # 스크롤 이동 대상 앵커 ID (첫 번째 H1의 영어 ID 또는 #what)
stats:                     # 히어로 영역 통계/주요 지표 칩 (최대 4개)
  - value: "약 10분"
    label: "설치 소요 시간"
  - value: "3가지"
    label: "운영체제 지원"
done:                      # 가이드 하단 피날레 최종 격려 섹션 정보
  title: "🎉 설치 완료! 이제 시작해봐요"
  subtitle: "아래 명령어들로 다양한 기능을 직접 탐험해 보세요!"
  ctaLabel: "공식 문서 바로가기"
  ctaUrl: "https://example.com"
footer:                    # 맨 하단 푸터 기재 사항
  - "이 가이드는 공식 사양을 기반으로 제작되었습니다."
  - "공식 문서: [example.com](https://example.com)"
---
```

### 🎨 테마 스타일 키 프리셋 (`style` 필드)
대상 도구의 성격에 따라 다음 10개 중 하나를 정확히 매핑하여 지정하십시오:
* `ai-chat` : AI 챗봇 & 어시스턴트 테마 (그린, HSL Tailored)
* `ai-dev` : AI 개발 및 코딩 도구 테마 (오렌지)
* `knowledge` : 지식 관리 및 문서 도구 테마 (퍼플)
* `productivity` : PC 유틸리티 및 생산성 테마 (블루)
* `creative` : 크리에이티브 생성 AI 테마 (마젠타)
* `security` : 보안 및 웹 인프라 테마 (사이언)
* `finance` : 재테크 및 금융 도구 테마 (골드)
* `future` : 미래 기술 & 트렌드 테마 (네온 라임)
* `enterprise` : 비즈니스 솔루션 테마 (슬레이트)
* `media` : 소셜 미디어 및 영상 제작 테마 (로즈)

## 2. H1 / H2 헤더 및 슬라이드 분할 규칙

파워포인트 변환기(`md-to-pptx.mjs`) 및 가로형 횡 슬라이더 빌더(`build-presentation.mjs`)는 마크다운을 H1 단위로 섹션을 쪼개고, 각 H2를 개별 카드 슬라이드로 자동 배치 및 높이를 계산합니다. 또한, Marp 호환을 위한 `---` 수평선 기준 슬라이드 분할을 동시에 완벽 지원합니다.

1. **H1 개수 제한**: 전체 마크다운 문서 내의 H1(`# `) 개수는 **4개 ~ 최대 6개 내외**로 통제하십시오.
2. **수평선 분할 활용**: H1/H2 구조를 유지하면서도 내용상의 추가 슬라이드 분리가 필요할 땐 `---` 수평선을 마크다운 중간에 넣어 수동으로 슬라이드 면을 분할하십시오. 헤더 분할과 수평선 분할이 공존해도 빈 슬라이드가 중복으로 남지 않도록 세이프 필터링이 적용되어 있습니다.
3. **H2 카드 배치**: 각 H1 하위에는 **H2(`## `)를 최대 3개 이하**로 배치하십시오. H2가 너무 많으면 파워포인트 슬라이드가 카드 높이 초과 경고를 유발하거나 가로 슬라이더 가독성이 복잡해집니다.
4. **일반 본문 단정화**: 숏코드 외에 마크다운 일반 텍스트 본문(문장, 일반 리스트)은 최소화하고, 모든 중요한 설명과 비교 분석 정보는 숏코드 블록 내부에 정밀 격리하십시오.

## 3. 자료 분석 기반 숏코드 적소 배치 규칙

입력된 원본 자료의 성격에 따라, Creative Spark v1.1 표준 숏코드를 적소에 조립하십시오.

### A. 가로 다단 격자형 숏코드 5종의 배치 논리
* **요금제, 플랜, 구독 서비스 비교** ➡️ **`plan-grid`**
  - *특징*: `featured: "true"` 또는 `color` 지정을 통해 카드를 시각적으로 도드라지게 표현합니다.
* **도구의 주요 기능 특징, 강점 소개** ➡️ **`feature-grid`**
  - *특징*: 아이콘이 제목 좌측에 인라인 배치되며, 우측 상단 배지 태그를 활용합니다.
* **대안 및 유형별 비교 분석, 장단점 나열** ➡️ **`compare-grid`**
  - *특징*: 카드 하단에 옅은 색의 메모란(`meta` 매핑)이 자동으로 연출되어 부연 설명에 제격입니다.
* **일반 텍스트 단 구성, 예제 코드/단축키 다단 나열** ➡️ **`columns-grid`**
  - *특징*: 군더더기 장식을 제거하고 3열/4열 다단 배치로 본문 가독성을 높입니다.
* **소요시간, 단계 수, 백분율 등 수치형 지표 강조** ➡️ **`stat-grid`**
  - *특징*: `icon` 영역에 거대하고 수려한 세리프(Serif) 수치(예: "95%", "10분")를 정렬 표기합니다.

### B. 가변 컬럼 제어 (`cols=N`) 적극 반영
* 상기 격자형 숏코드 5종 선언 시, 카드의 개수에 맞춰 선언부 옆에 **`cols=3` 또는 `cols=4` 인수**를 지정하여 브라우저에서 격자 단수가 강제로 칼각 고정 정렬되도록 하십시오.
  * *예시*: `::: feature-grid cols=3` , `::: plan-grid cols=4`

### C. 프리미엄 3대 비주얼 특화 숏코드 탑재 논리
* **Obsidian 지식 성단, 둥둥 떠다니는 연결 노드 구성** ➡️ **`network-box`**
  - *매핑*: 첫 번째 아이템이 중앙 거대 코어가 되며, 뒤이은 아이템들이 유영하는 위성 노드가 됩니다.
* **VS Code, 가상 에디터 코드 시뮬레이션 창** ➡️ **`editor-box`**
  - *매핑*: `title`에 파일명, `tag`에 개발 언어, `desc`에 예제 코드 멀티라인을 배치해 행 번호를 자동 부여합니다.
* **GitHub 브랜치 흐름, 다선형 커밋 로그 트리 도식화** ➡️ **`git-flow`**
  - *매핑*: 브랜치 이름과 커밋 상세 팝오버 리스트(`meta`에 파이프 `|` 기호로 구분)를 연동 렌더링합니다.

### D. 슬라이드 로컬 스타일 오버라이드 숏코드 (`::: slide-config`)
* **용도**: 특정 슬라이드/페이지 장에서만 일시적으로 배경색(`bg`), 본문 글자색(`color`), 타이틀/본문 폰트 크기(`titleSize`, `bodySize`)를 개별 조율하고 싶을 때 사용합니다. 다음 분할 트리거(H1, H2, `---`)를 만나는 즉시 스타일이 기본값으로 리셋됩니다.
* *예시*:
  ```markdown
  ::: slide-config
  bg: "#0F172A"       # 해당 페이지 배경색 (Dark)
  color: "#FFFFFF"    # 해당 페이지 본문 글자색
  titleSize: 22       # 해당 페이지 타이틀 크기 임시 오버라이드
  :::
  ```

## 4. 6대 표준 키(Universal Schema) 스키마 규칙

숏코드 내부의 마크다운 리스트 속성은 오직 아래 정의된 **6대 표준 키**만을 사용하여 YAML-like 형식으로 선언해야 하며, 들여쓰기 4칸 및 멀티라인 리터럴(`desc: |`)을 엄격히 준수해야 합니다.

* `icon`: 항목별 이모지 혹은 stat-grid 용 거대 지표 문구
* `title`: 대제목 / 라벨 / 카드 제목
* `desc`: 본문 설명 (여러 줄일 경우 `desc: |` 형식을 사용하고 다음 줄에 4칸 들여쓰기 후 기재)
* `tag`: 강조 배지 라벨 텍스트
* `meta`: 부가 속성 (또는 파이프 `|` 기호로 연결된 목록 상세 사항)
* `color`: 개별 카드를 브랜드 칼라로 테두리/뱃지 발광 강조하고 싶을 때 쓰는 헥스 컬러 (예: `"#6366F1"`)
