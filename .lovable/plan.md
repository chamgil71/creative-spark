# PPT 편집 가능 버전 추가

기존 이미지 캡처 방식 PPT는 그대로 두고, **별도 버튼**으로 네이티브 변환 PPT를 추가합니다. 다운로드한 PPT의 텍스트·표·도형을 PowerPoint에서 직접 수정할 수 있게 됩니다.

## 헤더 버튼 구성 (변경 후)

```text
[PPT (이미지)]  [PPT (편집)]  [PDF]  [새 탭]
```

- **PPT (이미지)** — 기존 `exportIframeToPptx`. 디자인 100% 재현, 편집 불가.
- **PPT (편집)** — 신규. 디자인은 단순화되지만 PowerPoint에서 자유 편집.
- **PDF** — 변경 없음.

## 신규 함수: `exportIframeToEditablePptx`

`src/lib/exportGuide.ts`에 추가. iframe 안의 가이드 DOM을 직접 순회하며 `pptxgenjs`의 네이티브 객체로 재조립합니다.

### 섹션 → 슬라이드 매핑 규칙

| HTML 요소 | PPT 변환 |
|---|---|
| `<h1>` (가이드 제목) | 표지 슬라이드 (다크 배경, 48pt 굵게) |
| `section.hero` | 1장 — 제목 + 부제 + 리드 문단 |
| `section.section` | H2 1개 = 슬라이드 1장 (내용이 길면 자동으로 N장 분할) |
| `section.done-section` | 마무리 1장 |

### 섹션 내부 블록 변환

| HTML | PPT 네이티브 |
|---|---|
| `<h2>` | 슬라이드 제목 — 32pt 굵게, 좌측 컬러바 도형 |
| `<h3>` | 소제목 — 20pt 굵게 |
| `<p>` | 본문 텍스트 박스 — 14pt, 좌측 정렬 |
| `<ul>/<ol>/<li>` | `addText`의 bullet 옵션 사용 |
| `.card`, `.feature-card`, `.tip-card` 등 | `addShape("roundRect")` 배경 + 내부 텍스트 박스 |
| `<table>` | `addTable` (헤더 행 강조) |
| `<pre>`, `<code>` 블록 | 회색 배경 도형 + 모노스페이스 텍스트 (Consolas) |
| `<blockquote>` | 좌측 컬러바 + 이탤릭 텍스트 |
| `<img>`, `<svg>` | base64로 인코딩해 `addImage` (가능한 경우) |
| 아이콘(lucide 등 SVG) | 텍스트 라벨로 대체 또는 생략 |

### 페이지 분할 알고리즘

각 `section.section`을 슬라이드 1장에 담되, 누적 콘텐츠 높이(추정치)가 슬라이드 본문 영역(약 6.0인치)을 넘으면 다음 슬라이드로 넘김. 카드/표는 항상 통째로 배치(중간에 자르지 않음).

### 색상·타이포

- 표지: 배경 `#0F172A`, 강조바 `#6366F1`
- 본문 슬라이드: 배경 흰색, 제목 `#0F172A`, 본문 `#334155`, 액센트 `#6366F1`
- 카드 배경: `#F8FAFC`, 테두리 `#E2E8F0`
- 한글 폰트: `Malgun Gothic` (헤더 굵게), 본문도 동일 / 코드: `Consolas`

### 좌표 기준 (16:9, 13.333 × 7.5인치)

- 좌우 마진 0.5인치, 상단 마진 0.6인치
- 제목 영역: y=0.4, h=0.8
- 본문 영역: y=1.4, h=5.6
- 페이지 번호: 우하단 (현재/전체)

## UI 변경 (`src/pages/GuideViewer.tsx`)

- `busy` 상태 타입에 `"pptx-edit"` 추가
- 새 버튼 추가, 라벨은 "PPT (편집)" / 기존 버튼은 "PPT (이미지)"로 변경
- 같은 `handleExport` 패턴 재사용, 신규 함수 호출

## 사용자에게 안내할 내용

- 편집 가능 PPT는 디자인이 단순화됨(그라디언트·블러·일부 레이아웃 손실)
- 대신 모든 텍스트/표/카드를 PowerPoint에서 직접 수정 가능
- 정밀한 디자인 보존이 필요하면 "PPT (이미지)" 사용
